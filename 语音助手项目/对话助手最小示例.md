# 一个关于AI对话助手的最小示例
本次完成AI对话助手的最小示例及其打包

## 环境准备
本示例依赖`llama_cpp_python`，我们需要先找到和自己当前环境相匹配的`llama_cpp_python`版本，并下载  
这里注意，我们要下载预编译版本的，在本地环境编译需要正确配置CMake，但是我这里试了几次都没成功，直接下编译好的版本吧！  
预编译版本[下载地址](https://abetlen.github.io/llama-cpp-python/whl/cpu/llama-cpp-python/)  
我们这里是python3.11，amd64系统，选择正确的版本下载到电脑上  
然后在该文件夹下启动虚拟环境，并执行：
```
pip install llama_cpp_python-0.3.19-cp311-cp311-win_amd64.whl
```
成功下载`llama_cpp_python`，这将是我们模型运行的引擎

## 模型下载
基于模型智能程度和设备配置，我们选择`Qwen2.5-7B-Instruct-GGUF-Q4_K_M`，GGUF是为llama.cpp准备的特殊模型文件格式，Q4_K_M则是量化标准  
将其下载后放到文件夹中，结构就像这样：
```
backend.py
models/
|_Qwen2.5-7b-instruct-q4_k_m.gguf
```

## 后端编写
`backend.py`
```python
import sys
import os

def get_resource_path(relative_path):
    """获取资源文件的绝对路径，兼容开发环境和 PyInstaller 打包后的环境"""
    if hasattr(sys, '_MEIPASS'):
        # 这是 PyInstaller 打包后的临时目录
        return os.path.join(sys._MEIPASS, relative_path)
    # 这是开发时的目录（相对于当前脚本的路径）
    return os.path.join(os.path.abspath("."), relative_path)

from llama_cpp import Llama

# 1. 初始化模型（只加载一次，这行代码运行后模型就常驻内存了）
print("正在加载模型，请稍候...")
model_path = get_resource_path("models/qwen2.5-7b-instruct-q4_k_m.gguf")  # 这里填你的模型文件路径
llm = Llama(
    model_path=model_path,
    n_ctx=4096,          # 上下文窗口大小，控制能记住多少对话历史，2048 纯 CPU 够用
    n_threads=12,         # 这里填你的 CPU 物理核心数（比如 4核就填4，8核就填8）
    n_batch=1024,         # 批处理大小，影响速度，512 是平衡值
    verbose=True,       # 设为 False 可以不看底层繁杂的日志，设为 True 方便调试
    chat_format="qwen"
)
print("模型加载完成！")

# 2. 设定阿塔尼斯的人设 (System Prompt)
system_prompt = """你是阿塔尼斯（Artanis），是《星际争霸》宇宙中的达拉姆大主教，星灵族的领袖。
- 你的性格高贵、威严、富有荣誉感，但也关心同伴。
- 你说话的风格庄重，偶尔会提及"卡拉"、"艾尔"、"为了艾尔"（En Taro Adun! / En Taro Tassadar!）等星灵的专有名词。
- 请用中文与用户对话，保持你的角色设定。
- 用户是一名圣堂武士，你要对用户展现你耐心、可靠的角色形象。
"""

# 3. 简单的对话循环
while True:
    user_input = input("\n你: ")
    
    if user_input.lower() in ["退出", "quit", "exit"]:
        break

    print("阿塔尼斯: ", end="", flush=True)
    
    # 4. 调用模型生成回复 (使用流式输出，体验更好)
    output = llm.create_chat_completion(
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_input}
        ],
        stream=True,  # 开启流式输出，像打字一样一个字一个字蹦出来
        temperature=0.7,  # 控制随机性，0.7 比较平衡
        max_tokens=512    # 最大生成长度
    )

    # 5. 打印流式输出
    for token in output:
        delta = token["choices"][0]["delta"]
        if "content" in delta:
            print(delta["content"], end="", flush=True)
```

## 打包准备
我们编写`backend.spec`文件，用`pyinstaller`将文件打包为exe  
这里的坑有很多，直接按照我这里给出的文件写即可，否则可能会漏掉`.dll`动态文件
`backend.spec`
```plaintext
# -*- mode: python ; coding: utf-8 -*-
block_cipher = None

# --------------------------
# 自动获取 llama_cpp 路径
# --------------------------
import llama_cpp
import os
LLAMA_CPP_PATH = os.path.dirname(llama_cpp.__file__)

# --------------------------
# 打包模型 + 依赖库
# --------------------------
datas = [
    ('models', 'models'),
    (os.path.join(LLAMA_CPP_PATH, 'lib'), 'llama_cpp/lib'),
]

# --------------------------
# 只打包 lib 里的 dll
# --------------------------
binaries = [
    (os.path.join(LLAMA_CPP_PATH, 'lib', '*.dll'), 'llama_cpp/lib'),
]

# --------------------------
# 必须导入的模块
# --------------------------
hiddenimports = [
    'llama_cpp',
    'llama_cpp.llama',
    'llama_cpp._ctypes_extensions',
    'numpy',
]

# --------------------------
# 固定配置
# --------------------------
a = Analysis(
    ['backend.py'],
    pathex=[],
    binaries=binaries,
    datas=datas,
    hiddenimports=hiddenimports,
    hookspath=[],
    runtime_hooks=[],
    excludes=[],
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    exclude_binaries=True,
    name='Artanis_LLM',
    console=True,
)

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    name='Artanis_LLM',
)
```
最后我们可以在`dist\Artanis_LLM`中找到我们的exe文件，点击即可用，非常的方便
