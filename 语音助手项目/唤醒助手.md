# 语音唤醒助手
该项目为ai阿塔尼斯项目的子模块，ai阿塔尼斯包含4个部分：
- 人机对话
- 语音转写
- 语音唤醒
- ai模拟音色

该项目为**语音唤醒**部分

## 调研
这个项目是接着语音转写项目做的，本来想也使用funASR，因为它的`paraformer-zh`模型转写效果还不赖，但是语音唤醒的本质是让一个模型长期挂在后台监听麦克风，当用户说出关键词时就触发信号。如果我们使用较大的模型，这会导致电脑一直处于卡顿之中，这并非我的本意也不是用户想要看到的。  
在调研过程中我找到几个语音唤醒模型：
- Porcupine/Picovoice(豪猪)，原来是一个开源项目，发展成公司业务了，听说效果很好，2年前还有大量资料教如何做语音唤醒项目，但是我这个时间节点，它已经全面紧缩个人用户，现在官网不允许非企业用户使用了。如果想要用GitHub上的开源项目，自定义一个唤醒词是需要自己训练的，这将带来一定的成本。
- Mycroft Precise，全开源项目，没有仔细去用，因为需要自己训练，既然如此为什么不用鼎鼎大名的豪猪？
- openWakeWord，未用过，资料显示可以用Google Colab一键训练自定义唤醒词，~~也许好用，也许不好用……~~

搞来搞去，最后也没搞出个东西来，求助ai大人，结果它说阿里的funASR下有一个子项目damo(达摩)，是做语音唤醒的，结果到modelscope上看的时候已经下架了（为什么！？）  
到这里我费了许多心力还没有搞出一个最小示例，已经释怀了……我释怀的时候，脑海里自动播放《关羽之歌》，然后我就想起我好像见过一个类似语音唤醒的项目，只不过是检测关键词的，也是Github开源！  
> ”也许，这就是这场战斗的胜负手……“  
感谢新3国，感谢自动玩梗器(AutoMemeDetector)作者：[@依然匹萨吧](https://space.bilibili.com/6297797)，当时发现这个项目很有意思专门去GitHub上看了源码，现在用上了，我们使用同款轻量级语音转写模型`Vosk`。

## 实验过程
依旧启动虚拟环境`conda activate funasr_env`  
安装依赖：
```
pip install vosk pyaudio numpy
```
手动下载模型，[下载地址](https://alphacephei.com/vosk/models/vosk-model-small-cn-0.22.zip)  
运行下面的python代码，根据转写结果来看，`vosk-model-small-cn`速度很快，效果勉强能看，`vosk-model-cn`效果较好，但是加载时间较长，我们最终选择small型号。  
我们可以自己测试一下，看看模型会把我们的话识别成哪些语句，可以在规则中把这些误识别的语句也添加为唤醒词，这样可以增加成功率。  
我这个代码同时还能将转写后的文字转换为拼音，再同时用拼音和文字进行匹配，进一步增加容错！(注意先`pip install pypinyin`)  
```python
import vosk
import pyaudio
import json
import pypinyin
import sys
import os

# --------------------------
# 核心修改：自适应模型路径（开发环境/打包后exe环境通用）
# --------------------------
if getattr(sys, 'frozen', False) and hasattr(sys, '_MEIPASS'):
    # 情况1：如果是打包后的exe在运行
    # PyInstaller会把模型解压到临时的_MEIPASS目录
    BASE_PATH = sys._MEIPASS
else:
    # 情况2：如果是开发环境直接运行py脚本
    BASE_PATH = os.path.dirname(os.path.abspath(__file__))

# 拼接模型路径
MODEL_PATH = os.path.join(BASE_PATH, "models", "vosk-model-small-cn")
WAKE_WORDS = [
    "你好阿塔尼斯", "启动助手", "qi dong",
    "大主教", "英明神武","明灯",  # 中文简称
    "a ta ni si", "artanis","er ta ni shi","er kan ni si","er ta ni si","a ta ni shuo",
    "a ka ni shuo","a ka ni si","a ta yi si","a ta mi si",
    "da la mu","ba la mu","da zhu jiao","ming deng","ba zhu jiao"
]

print("正在加载Vosk模型...")
model = vosk.Model(MODEL_PATH)
rec = vosk.KaldiRecognizer(model, 16000)
rec.SetWords(True)
print("✅ 模型加载完成，正在监听唤醒词...")

FORMAT = pyaudio.paInt16
CHANNELS = 1
RATE = 16000
CHUNK = 4000

p = pyaudio.PyAudio()
stream = p.open(format=FORMAT, channels=CHANNELS, rate=RATE, input=True, frames_per_buffer=CHUNK)

def fuzzy_match_wake_word(text):
    text_pinyin = " ".join(pypinyin.lazy_pinyin(text)).lower()
    text_lower = text.lower()
    for wake_word in WAKE_WORDS:
        if (wake_word in text_lower) or (wake_word in text_pinyin):
            return wake_word
    return None

try:
    while True:
        data = stream.read(CHUNK)
        if rec.AcceptWaveform(data):
            result = json.loads(rec.Result())
            text = result.get("text", "").replace(" ", "")
            if text:
                print(f"识别内容：{text}")
                matched_word = fuzzy_match_wake_word(text)
                if matched_word:
                    print(f"🎉 检测到唤醒词【{matched_word}】！")
                    # --------------------------
                    # 关键：打印特定标记给Electron
                    # --------------------------
                    print("WAKE_WORD_DETECTED")
                    sys.stdout.flush() # 强制刷新缓冲区，确保Electron能立即收到
        else:
            pass

except KeyboardInterrupt:
    print("停止监听")
finally:
    stream.stop_stream()
    stream.close()
    p.terminate()
```
我这里得到的效果非常好，实在是太荣耀了！~~这就是对抗圣堂武士的下场！~~  

## 语音唤醒
想要真正集成到最终的项目中，我们还需要用Electron打包起来，这里ai大人给的建议是：先把后端python用`pyinstaller`打包成一个exe文件，测试该文件正常运行后再将整个项目用Electron打包为exe  
先用Vite创建Vue3项目：
```
npm create vite@latest kws-test -- --template vue-ts
cd electron-atanis-assistant
npm install
npm install -D electron electron-builder vite-plugin-electron vite-plugin-electron-renderer
npm install -D esbuild
```
项目结构应该如下：
```
kws-test/
├── electron/
│   └── main.ts          # 新增：Electron主进程代码
├── models/              # 新增：把你的Vosk模型文件夹放这里
│   └── vosk-model-small-cn/
├── wake_word_service/   # 新增：Python唤醒词脚本和打包后的exe
│   └── wake_word.exe    # 稍后用PyInstaller打包生成
├── src/
│   └── App.vue          # 修改：简单页面
├── vite.config.ts       # 修改：配置Electron
└── package.json         # 修改：配置入口和打包
```
需要编写`electron/main.ts`，具体见`main.ts`  
还需要配置`vite.config.ts`和`package.json`  
接下来将`wake_word_service.py`打包为exe：
```
pip install pyinstaller
cd wake_word_service
```
我们这里选择把模型和python文件打包在一起，这样就不用到处找了，先把models文件夹复制到wake_word_service中  
```
pyinstaller -w -F --add-data "models;models" -n wake_word wake_word.py
pyinstaller -F --add-data "models;models" --collect-data vosk --collect-binaries vosk -n wake_word wake_word.py
```
- `-w`代表不显示控制台窗口，想要试试是否成功可以先不加
- `-F`打包为单个exe文件
- `--add-data "models;models"`把models也打包进去，Windows系统这里是分号，Mac/Linux是冒号
- `-n`指定名字

> 这里需要用下面那个命令，因为打包时容易没有把vosk库自带的.dll动态链接库、内置资源文件打包，会报错的  

最终会在dist文件夹中产生一个exe文件，我们点击试试有没有问题，`build`文件夹下是打包的临时文件，可以删除了；.sepc是打包配置文件，可以不删  
现在将exe文件单独复制到wake_word_service文件夹下，我们准备开始全局打包

### 全局封装
按照上面给出结构构建好项目后，修改`vite.config.ts`和`package.json`（具体修改情况直接参考我的相关项目仓库）  
开发环境运行：`npm run electron:dev`  
打包成exe：`npm run electron:build`，打包完成后，会生成dist、dist-electron和release 3个文件夹，其中release中会有安装包  
注意:
> 我们要在虚拟环境下打包，否则会出现环境问题，并且要用管理员身份打开CMD，这样可以避免权限不够问题  
> 打包过程中需要到GitHub上下载Electron核心二进制文件，可能出现网络问题，请在项目根目录下创建一个文件`.npmrc`，并写入以下内容：
```
electron_mirror=https://npmmirror.com/mirrors/electron/
electron_builder_binaries_mirror=https://npmmirror.com/mirrors/electron-builder-binaries/
win_code_sign_mirror=https://npmmirror.com/mirrors/winCodeSign/
nsis_mirror=https://npmmirror.com/mirrors/nsis/
nsis_assets_mirror=https://npmmirror.com/mirrors/nsis-assets/
```
> 图标寻找可去[该网址](https://iconify.design/)，遇到ico图标转换问题，可以去[转换网站](https://convertio.co/zh/)  
> 一定要在`package.json`文件中配置“关闭一键安装”、“自定义路径”和“取消自动更新”，否则它会安装奇怪的程序到奇怪的地方
