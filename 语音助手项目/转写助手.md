# 语音转写助手
这是ai阿塔尼斯项目的一个实验子模块，ai阿塔尼斯项目包含：
- 语音唤醒
- 语音输入
- ai回复
- ai模拟角色音色、

本项目为“语音输入”部分

## 调研
调研了一下，语音输入也就是语音转写，需要把语音转换为文字，输入到输入框中。  
由于这是个人学习项目，没有足够的经费，所以不考虑调用api直接实现语音输入，而是考虑一个语音转写功能。  
调研过程中找到几个还不错的模型，本来打算用Whisper，但是使用过程中发现几个问题：
它虽然按照使用场景分为几个模型：small、base、medium、large……，但是小模型转写能力惨不忍睹，大的模型加载时间长且处理1次3到5s的语音的时间也需要9到15s，这对于一个语音助手是不合格的  
最终我使用了国产阿里的模型funASR系列的`paraformer-zh`，首先中文场景就更加适配；其次它的速度非常快，加载和处理速度都很快（即使在我这种纯CPU设备）  
最终该子模块的技术栈如下：
前端：Vue3+TypeScript
后端：Python+FastAPI

## 开始实验
### 环境准备
依赖较多，建议创建一个虚拟环境，当然，你需要先确保自己电脑上有Miniconda或者Anaconda，并且需要添加bin到系统环境变量，否则你需要在Anaconda Prompt中运行下面的命令，这显然是很麻烦的，我这里使用的是Anaconda，不过区别也不大：
```
conda create -n funasr_env python=3.11 -y
```
这个虚拟环境默认安装到`C:/Users/你的用户名/anaconda3/envs`  
激活虚拟环境：
```
conda activate funasr_env
```
安装依赖：
```
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
pip install funasr
```
如果安装慢可以上镜像（未验证）：`pip install funasr -i https://pypi.tuna.tsinghua.edu.cn/simple`  

### 最小示例
我们准备一个文件夹`my_audio`，里面随便放点音频文件（WAV格式是最好的，MP3、m4a也可以）  
然后在外层来一个python代码：
```python
import os
from funasr import AutoModel

def transcribe_audio_folder(folder_path):
    # 1. 加载 FunASR 模型（paraformer-zh 是轻量级中文模型，适合 CPU）
    print("正在加载模型，请稍候（首次运行会自动下载模型，约 200MB）...")
    model = AutoModel(model="paraformer-zh")
    
    # 2. 遍历文件夹下的音频文件
    audio_extensions = ('.wav', '.mp3', '.m4a')  # 支持的音频格式
    for filename in os.listdir(folder_path):
        if filename.lower().endswith(audio_extensions):
            audio_path = os.path.join(folder_path, filename)
            print(f"\n正在处理：{filename}")
            
            # 3. 调用模型进行转写
            result = model.generate(input=audio_path)
            
            # 4. 提取并打印转写结果
            # result 是一个列表，第一个元素的 'text' 字段就是转写文字
            if result and 'text' in result[0]:
                text = result[0]['text']
                print(f"转写结果：{text}")
            else:
                print("未识别到有效内容")

if __name__ == "__main__":
    # 在这里修改你的音频文件夹路径（注意 Windows 路径用双反斜杠 \\ 或单斜杠 /）
    YOUR_AUDIO_FOLDER = "D:\\test_audios"
    
    if os.path.exists(YOUR_AUDIO_FOLDER):
        transcribe_audio_folder(YOUR_AUDIO_FOLDER)
    else:
        print(f"错误：文件夹 {YOUR_AUDIO_FOLDER} 不存在，请检查路径！")
```
直接在虚拟环境中执行`python test.py`就可以了。  
如果在cmd上看见了转写的内容（音频最好声音大一点，并且说话不能含糊），就证明成功了，如果觉得不行可以自己换模型。这个代码是会自动下载魔搭上的模型的，下载路径：`C:\Users\用户名\.cache\modelscope\hub\models\iic`（`C:\Users\用户名\anaconda3\envs\funasr_env\Lib\site-packages\funasr\models`是源代码，定义了模型网络的架构~~有多少层等~~、前向传播逻辑、解码算法）  

### 语音转写
本来看见这个paraformer有流式模式，就想着试一下，但是在纯CPU环境下出现轻微卡顿，并且检测效果也不好，不如音频文件检测。  
我觉得还是前端录制音频，再传输临时文件给后端，后端转写后再将文字结果传回前端，这样的方式比较适合我们的项目。注意到每次加载模型  
这个来回传输需要WebSocket和FastAPI支持：
```
pip install fastapi uvicorn websockets
```
后端、前端代码见`backend_fast.py`、`VoiceTranscriber.vue`。  
注意：
> 应该让模型在启动时就加载，在后台待命，否则一次的处理时间会较长  
> 前端需要对过短的录音直接丢弃，否则模型会将其视为非法文件
