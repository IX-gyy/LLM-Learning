里面的系统太旧了，而且没有Windows这种自动更新机制，居然必须要**刷机**才能更新系统

从来没刷过机的也是成功第一次了，不过他们管这个过程叫*烧写*  
宿主机：我的手提电脑`银骑士`  
开发板：`Jetson Orin NX CLB`  
开搞！

## 1.下载NVIDIA SDK Manager
在宿主机上下载这个东西和一个USB Driver，这个过程需要登录英伟达账号（如果使用命令行下载，在软件内部还是需要登录英伟达账号，所以建议直接在[官网](https://developer.nvidia.com/sdk-manager)上下载）  
AI大人给出的教程似乎是old版本的，提示必须要在Linux系统上下载，不过英伟达官方已经更新了适配Windows的下载器（本机上有WSL2就行）  

## 2.开发板进入烧写状态
下载过程中也别闲着，将开发板调整为烧写状态  
首先断电（我震惊地得知这个板子居然没有开关机按钮，只能通过直接断电通电来开关机）  
然后移除连接板子的所有USB设备，并根据说明书使用电线将板子短接  
使用转换线将板子和宿主机连接，通电  

## 3.开始烧写
宿主机也移除所有的USB连接设备，只和开发板连接（之前连接了设备，中途报错1次，不知道是哪一个设备导致的，索性全部拔了）  
宿主机打开SDK Manager，登录英伟达账号，按照软件流程走，记得选择系统版本  
它会先在宿主机上安装，然后再传输到开发板上（为什么不直接把安装地址设置在开发板上(*Φ皿Φ*)）  
中途会要求设置Linux账号和密码，记得看着点，否则进度不动浪费时间  

## 4.烧写完成
完成后自动提示，确定结束后将开发板断电，并拔出短接电线和转换线，重新连接相关USB设备  
再次开机，可以发现刷机了，终端输入


## 5.开启ssh远程连接
依旧踩坑:(  
首先下载ssh server `sudo apt-get install openssh-server`  
使用vim修改配置文件 `sudo vim /etc/ssh/sshd_config`  
vim的使用方法如下：
```
按i进入编辑模式
编辑完毕后输入 :wq 保存并退出
```
我们要找到这一行字：`#PermitRootLogin prohibit-password`，将它替换为`PermitRootLogin yes`，代表允许root用户使用密码登录  
重启系统或ssh服务，并开启开机自启动：
```
sudo /etc/init.d/ssh stop
sudo /etc/init.d/ssh start
sudo service ssh restart
sudo systemctl enable ssh
```
查询一下是否设置成功：输入`sudo systemctl status ssh`，如果出现`active(running)`则说明成功了！  
然后我被狠狠信息差了，因为刷机之前连接过，所以SSH为了防止中间攻击使用了**指纹**机制，在第一次连接时两个机器生成了特殊编码，现在刷机了就不认以前的编码了  
最后在本机的`.ssh`文件夹中找到了`known_hosts`文件，将其中以前的指纹删除，VSCode再次连接时会自动生成新的指纹
