## 2.安装docker
此处使用清华镜像源：
```
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io
```
安装NVIDIA Toolkit
```
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
sudo apt update
sudo apt install -y nvidia-container-toolkit
sudo systemctl restart docker
```
验证，如果正常输出GPU信息，则准备就绪`docker run --rm --runtime=nvidia nvcr.io/nvidia/l4t-base:r36.4.0 nvidia-smi`

## 3.模型准备
主要模型：
- `Qwen/Qwen3-VL-2B-Instruct`
- `Qwen/Qwen3-VL-8B-Instruct`
- `cyankiwi/Qwen3-VL-2B-Instruct-AWQ-4bit`
- `cyankiwi/Qwen3-VL-8B-Instruct-AWQ-4bit`

下载模型的3种方法：
### I 使用python代码下载
大创项目多次试验，old含金量，虽然很麻烦而且速度一般，但是成功率高  
将下面的代码命名为`download.py`，然后在命令行运行`python download.py`，可以将模型下载到当前文件夹
```python
import os
from huggingface_hub import snapshot_download

os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"

snapshot_download(
    repo_id="Qwen/Qwen3-VL-2B-Instruct",
    local_dir="./Qwen3-VL-2B-Instruct",
    local_dir_use_symlinks=False,
    resume_download=True
)
print("下载完成！")
```
### II 使用huggingface官方工具
`huggingface-cli` 是Hugging Face官方提供的命令行工具，自带完善的下载功能。使用方法如下：  
1. 安装依赖  
```
pip install -U huggingface_hub
```
3. 设置环境变量  
```
export HF_ENDPOINT=https://hf-mirror.com
```
3.1 下载模型  
```
huggingface-cli download --resume-download Qwen/Qwen3-VL-2B-Instruct --local-dir ./Qwen3-VL-2B-Instruct
```
3.2 下载数据集  
```
huggingface-cli download --repo-type dataset --resume-download wikitext --local-dir wikitext
```

### III 使用hfd
`hfd`是hf-mirror.com开发的 huggingface 专用下载工具，基于成熟工具`aria2`，可以做到稳定高速下载不断线。  
1.下载hfd
```
wget https://hf-mirror.com/hfd/hfd.sh
chmod a+x hfd.sh
```
2.设置环境变量
```
export HF_ENDPOINT=https://hf-mirror.com
```
3.1 下载模型  
```
./hfd.sh Qwen/Qwen3-VL-2B-Instruct --local-dir ./Qwen3-VL-2B-Instruct
```
3.2 下载数据集  
```
./hfd.sh wikitext --dataset
```

## 4.拉取docker镜像
查看系统版本，下载docker要匹配
```
cat /etc/nv_tegra_release
```
根据查到的系统类型，选择我们需要的docker镜像，这里我是
```
# R36 (release), REVISION: 5.0, GCID: 43688277, BOARD: generic, EABI: aarch64, DATE: Fri Jan 16 03:50:45 UTC 2026
# KERNEL_VARIANT: oot
TARGET_USERSPACE_LIB_DIR=nvidia
TARGET_USERSPACE_LIB_DIR_PATH=usr/lib/aarch64-linux-gnu/nvidia
```
所以我们选择：  
vllm镜像（先看看能否跑通，再试试llama）
```
docker pull ghcr.io/nvidia-ai-iot/vllm:r36.4-tegra-aarch64-cu126-22.04
docker pull ghcr.io/nvidia-ai-iot/llama_cpp:r36.4-tegra-aarch64-cu126-22.04
```
启动vllm服务容器：
创建容器并挂载模型目录（注意将 /home/tp/Workspace 替换为你实际的路径，如 /home/yourname/Workspace）  
这里将开发板中的模型通过`:`映射到docker容器`vllm_deploy`中的`workspace/model`（注意并没有被装到一个文件夹中，而是全部装在model中，下面已经修正）
```
docker run -itd \
  --name vllm_deploy \
  --runtime nvidia \
  --network host \
  --ipc host \
  -e NVIDIA_VISIBLE_DEVICES=all \
  -e NVIDIA_DRIVER_CAPABILITIES=compute,utility \
  -v /home/nx/lt/models/Qwen3-VL-2B-Instruct:/workspace/models/Qwen3-VL-2B-Instruct \
  ghcr.io/nvidia-ai-iot/vllm:r36.4-tegra-aarch64-cu126-22.04
```
如果创建失误可以销毁容器重新创一个：
```
# 停止容器
docker stop vllm_deploy

# 删除容器（会清理残留进程）
docker rm vllm_deploy

# 确认没有残留
docker ps -a | grep vllm_deploy
```

## 5.进入容器并启动模型服务
先确定docker已经启动
```
sudo docker start
```
进入容器：
```
docker exec -it vllm_deploy bash
```
激活虚拟环境（镜像内置）：
```
source /opt/venv/bin/activate
```
启动vllm服务：
```
vllm serve /workspace/model \
  --max-model-len 16384 \
  --gpu-memory-utilization 0.75 \
  --port 8000
```
再开一个终端测试是否成功启动：`curl http://127.0.0.1:8000/health`，如果返回`{"status":"ok"}`则正常

## 6.开始测试！
终于开始测试了捏，参考下面的命令行
```
vllm bench serve \
  --backend openai-chat \
  --model /workspace/model \
  --host 127.0.0.1 \
  --port 8000 \
  --endpoint /v1/chat/completions \
  --dataset-name hf \
  --dataset-path lmarena-ai/vision-arena-bench-v0.1 \
  --hf-split train \
  --num-prompts 128 \
  --request-rate inf \
  --max-concurrency 1
```
--backend openai-chat：使用 OpenAI 兼容的聊天接口  
--model：指定模型路径（仅用于日志，不影响服务）  
--host 和 --port 指向正在运行的服务  
--dataset-name hf：从 Hugging Face 加载数据集  
--dataset-path lmarena-ai/vision-arena-bench-v0.1：这是一个视觉问答数据集，包含图片和问题  
--num-prompts 128：发送 128 条请求  
--request-rate inf：无限请求率，即尽可能快地发送（压力测试）  
--max-concurrency 1：最大并发数为 1，即一次只处理一个请求

### 在 llama.cpp 容器中编译并转换 GGUF
以下步骤都在 `llama_cpp_deploy` 容器中完成。
#### 拉代码
如果没有代码：
```bash
cd /workspace
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
```
如果已经有了：
```bash
cd /workspace/llama.cpp
git pull
```
#### 打 patch，让 mmproj 也能量化
编辑：
```bash
vim src/llama-quant.cpp
```
找到：
```cpp
quantize &= name.find(".position_embd.") == std::string::npos;
```
在下面加一行：
```cpp
quantize &= name.find(".patch_embd.") == std::string::npos;
```

#### 编译
```bash
rm -rf build
cmake -B build -G Ninja -DGGML_CUDA=ON -DCMAKE_CUDA_ARCHITECTURES=87 -DBUILD_SHARED_LIBS=OFF
cmake --build build --config Release --parallel
```

#### 转 GGUF
主模型转换：
```bash
python convert_hf_to_gguf.py /path/to/hf_model --outtype bf16 --outfile /path/to/gguf
```
例如：
```bash
python convert_hf_to_gguf.py /workspace/models/Qwen3-VL-8B-Instruct --outtype bf16 --outfile /workspace/models/Qwen3-VL-8B-Instruct.gguf
```
mmproj 转换：
```bash
python convert_hf_to_gguf.py /path/to/hf_model --outtype bf16 --mmproj --outfile /path/to/gguf
```
例如：
```bash
python convert_hf_to_gguf.py /workspace/models/Qwen3-VL-8B-Instruct --outtype bf16 --mmproj --outfile /workspace/models/MMProj-Qwen3-VL-8B-Instruct.gguf
```

#### 量化
```bash
./build/bin/llama-quantize /path/to/original_gguf /path/to/quant.gguf Q4_K_M
```
例如：
```bash
./build/bin/llama-quantize /workspace/models/Qwen3-VL-8B-Instruct.gguf /workspace/models/Qwen3-VL-8B-Instruct-Q4_K_M.gguf Q4_K_M
```
